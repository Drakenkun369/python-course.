Object-Oriented Programming (OOP) is a programming paradigm that uses objects and classes. In OOP, a class defines the structure and behaviors of objects, and objects are instances of a class.

1. Classes and Objects

A class defines the blueprint for an object. It contains methods (functions) and attributes (variables). An object is an instance of a class.

2. Inheritance

Inheritance allows one class (child) to inherit the properties and methods from another class (parent). This enables code reuse and hierarchical classification.

3. Polymorphism

Polymorphism allows different objects to be treated as instances of the same class through inheritance. It enables overriding of methods in the child class.

4. Encapsulation

Encapsulation restricts direct access to an objectâ€™s attributes and methods, ensuring data protection by using getters and setters.

5. Abstraction

Abstraction is the concept of hiding the complex implementation details and showing only the necessary features of an object to the user.
